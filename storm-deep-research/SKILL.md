# STORM 深度研究写作 (research-then-write)

把"写长文"拆成 **先研究、后写作**。三个支柱(STORM 的核心洞察)：
1. **多视角** — 用不同角色/利益相关者的视角来驱动提问，保证广度与深度。*直接让模型"自己想问题"效果差*，必须用视角约束提问过程。
2. **源头接地 + 引用** — 每条非常识论断都必须 grounded 在检索到的真实来源上，带内联引用；**没来源就不写**，杜绝臆造。
3. **大纲优先** — 先搭层级结构，再往里填内容，保证逻辑连贯。

工具：`WebSearch` / `WebFetch`(主)，`Read`(本地资料)；按需多轮检索。

---

## 阶段 A · 预写作 (Pre-writing)

### A1 视角发现 (Perspective Discovery)
- 先对主题及相关主题做背景检索，归纳 **3–6 个互补视角/角色**（如：技术原理、商业/市场、监管/伦理、批评者/空方、历史脉络、终端用户、竞争对手）。
- 视角要能覆盖**争议面**，不是同义重复。

### A2 多视角提问 (Perspective-Guided Question Asking)
- 对每个视角，**模拟"维基写作者 × 该领域专家"的多轮对话**：写作者站在该视角连续追问，专家用检索到的来源作答，答案再激发下一问。
- 每轮闭环：问题 → 转检索 query → WebSearch/WebFetch 取证 → 记录「论点 + 来源URL + 关键数据」。追问到信息饱和为止。

### A3 来源池 (Retrieval / Grounding)
- 维护一个**来源池**：每条 = `{编号, 标题, URL, 关键事实/数据, 可信度}`。一手/权威来源优先，去重。
- **来源冲突时并存并标注分歧**，不武断取一。

## 阶段 B · 大纲 (Outline Generation)
- 综合来源池生成**层级大纲**(章/节)，逻辑递进、覆盖各视角与争议点。
- 每个小节标注"由来源池哪些编号支撑"；若某节缺证据 → **回 A2 补检索**，别硬写。

## 阶段 C · 写作 (Article Generation)
- **逐节填充**：用来源池内容写，每个事实/数据/论断带**内联引用 `[n]`**(n=来源编号)。
- 不放无引用的断言；不引用未真正打开核实过的来源(WebFetch 核过)。
- 中立、多视角并陈；有分歧处写明"X 认为…而 Y 认为…"。

## 阶段 D · 润色 (Polishing)
- 去重/合并重复信息；通读理顺过渡；补开头**导语(lead)** 与必要小结。
- 末尾汇总 **References**（编号 → 标题 + URL），与正文 `[n]` 一一对应、无悬挂引用。

---

## 可选 · Co-STORM 模式（复杂/有争议主题，或需人来共同引导）
- **专家圆桌**：起多个专家 persona 各自基于来源发言 + 一个**主持人(Moderator)** 抛出尖锐、推进性的问题打破盲区。
- **思维导图(Mind Map)**：维护一个**动态更新的概念层级**，把收集到的信息挂上去——既是人机共享的认知空间，也用于发现尚未覆盖的分支。
- **回合管理(turn policy)**：专家/主持人轮流发言推进；用户可旁观或随时介入 steer 方向。
- 用于让对立观点充分碰撞、或主题大到需要边研究边和用户对齐。

## 质量准则
- 每条非常识论断**必有引用**；引用的来源必须真读过。
- 至少覆盖 A1 列出的全部视角；**争议必须双呈**。
- 先大纲后正文；大纲暴露的证据缺口必须回补。
- 标注**不确定性与时效**（信息日期、是否一手）。

## Don't
- 别在没检索/没引用时凭记忆写"事实"。
- 别只从单一视角写（STORM 的本意就是反对单视角）。
- 别引用未打开核实的链接、别编造 URL。
- 别跳过大纲直接长篇堆砌。
- 来源冲突别强行二选一；并存 + 标注。

## 输出结构
1. **标题** + 一段**导语(lead)** 概述全文。
2. **正文**：按大纲分节，处处带 `[n]` 引用。
3. （Co-STORM 时）附「关键分歧 / 圆桌要点」小节。
4. **References**：编号 → 标题 + URL。
