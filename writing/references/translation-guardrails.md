# 翻译与保真改写护栏

先保证结构和信息一一对应，再把译文修成自然、准确、简洁的目标语言。除非用户允许自由改写，否则以下规则强制生效。

## 保持结构

- 保留标题层级、段落顺序、列表层级、表格列数、引用层级、空行和有意义的换行。
- 不合并或拆分段落、列表项和表格单元；只在单元内部调整句序。
- 保留原文的论证顺序、语气强度、重复、条件、例外和不确定性。

## 保持信息

- 不新增背景、例子、评价、立场、推断、总结或译者注。
- 不删除事实、数字、限定、否定、因果关系和承载意义的重复。
- 不把正文缩成摘要，不把描述增强为断言，不把相关性增强为因果性。
- 发现原文明显矛盾或疑似错误时，忠实翻译；只有用户要求校订时才另行指出。

## 保护不可改动内容

原样保留以下内容：

- fenced code block、缩进代码块和内联代码；
- URL、email、路径、命令、参数、包名、类名、函数名和变量名；
- 模板占位符、配置键、ID、哈希、版本号和数学公式；
- Markdown 结构符号、链接目标和表格对齐标记。

只翻译 Markdown 链接的可见文字。除非用户要求本地化数值，否则保留精确数字和单位，不擅自换算。

## 处理术语和专名

- 使用目标语言中稳定、通行的译名。
- 没有稳定译名的专名保留原文，不自造译名。
- 技术术语以准确为先；读者可能不熟悉时，可在首次出现处写“译名（原文）”。
- 同一概念使用同一译法。用户提供术语表时，以术语表为准。

## 两遍修订

第一遍逐段翻译，核对所有信息点和不可改动内容。第二遍只在段内调整词序、句式和连接，使目标语言自然，并执行以下检查：

- 行动者、时间、否定范围和指代是否明确；
- 术语是否一致，数字与单位是否准确；
- 是否残留源语言语序、空泛名词化或多余被动；
- 是否误加原文没有的路标、标题、强调或结论；
- 版式是否与原文对应。

默认只输出译文，不附分析、说明或自检记录。
