## 8. 会话与更新协议

### 0→1

先做目标、边界、假设和成功标准的理解回显；共识写入本文件后，再切首个任务包。

### 1→N

新会话依次读取：

1. `program.md`
2. `codemap.md`
3. 当前任务包
4. 最近相关证据与提交

然后说明当前目标、边界、任务状态、漂移、风险和下一 Checkpoint。

### 更新规则

- 跨任务仍成立的决定写入本文件。
- 当前实现结构写入 `codemap.md`，不复制到本文件。
- 单次推进、Checkpoint 和证据写入当前任务包。
- 发现文档与实现冲突时，标记漂移并处理，不静默猜测。
